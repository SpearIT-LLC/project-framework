## Decision Points

### When to Move Features from Backlog to Sprint

Features move from [planning/backlog](backlog/) to [work/todo](../work/todo/) when:
1. Clear user need identified
2. Dependencies satisfied
3. Time/resources available
4. Fits current version goals

### When to Defer Features

Features stay in backlog if:
- Speculative (no proven need)
- Dependencies not ready
- Complexity outweighs benefit
- Better approaches may emerge
