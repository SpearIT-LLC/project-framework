**Semantic Versioning Guide:**

Given a version number MAJOR.MINOR.PATCH (e.g., 1.2.3):

- **MAJOR** version (1.0.0 → 2.0.0): Breaking changes, incompatible API changes
- **MINOR** version (1.0.0 → 1.1.0): New features, backward-compatible additions
- **PATCH** version (1.0.0 → 1.0.1): Bug fixes, backward-compatible fixes

**Keep a Changelog Categories:**

- **Added:** New features
- **Changed:** Changes to existing functionality
- **Fixed:** Bug fixes
- **Deprecated:** Soon-to-be removed features
- **Removed:** Removed features
- **Security:** Security vulnerability fixes
